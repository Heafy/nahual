Motion debug parameters

Turn on the "Motion debug" box under the camera to see live values, which show how the app tracks hand movement.

  • raw motion — How much your hand moved between the two most recent camera frames. Higher number means faster movement. It reacts instantly, so it flickers up and down a lot as you move.
  • smoothed — Averaged version of raw motion. This is the number the app actually uses to decide when a moving letter begins and ends.
  • start/stop — Two fixed limits. When smoothed motion rises above the start value, the app begins recording a moving letter; when it drops below the stop value, the recording ends and is read.
  • state / buf — "state" is what the app is doing now: IDLE (waiting) or RECORDING (capturing a moving letter). "buf" is how many frames it has captured so far in the current recording.
